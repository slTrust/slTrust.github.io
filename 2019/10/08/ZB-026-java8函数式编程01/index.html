ZB-026-java8函数式编程01

为什么要使用函数式编程

  • 减少工作量
  • 提高效率
  • 减少bug

条件筛选用户实例

可以看到非常麻烦!!!代码非常的重复

先用接口来实现这个功能

不同条件都去实现一个类太麻烦

简化

Predicate

回忆高中数学的函数概念y = f(x)

从 x 到 y 的映射

Predicate来简化

  • test方法实际就是 User 到 boolean 的映射

lambda表达式

为什么这里可以把 Predicate 简化为 lambda表达式呢?

  • 说白了 test 就是一个函数
  • 任何满足满足从 T 到 boolean 值的映射 都可以自动被转换为一个函数接口

什么是函数接口？

  • Predicate 源码上有一个注解@FunctionInterface 它只是一个标记，不是必要的即使去掉也可以正常工作

三种方式 Predicate

  • lambda 表达式
    • user -> user.name.startsWith("王")

  • static 方法

  • 实例方法 instance method

函数接口详解

什么东西可以自动被转换为函数接口呢？

  • 任何只包含一个抽象方法的接口都可以被自动转换为函数接口
    • 如Predicate 只有一个 test 是抽象方法，其他都有方法体
    • 如我们自己实现一个
