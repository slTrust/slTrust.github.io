使用Next.js

创建项目

CRM开始 改内容

创建页面

  • pages/posts/first-post.js
  • 为了更好的类型提示 ,安装类型声明文件
  • 运行yarn dev
  • 如何访问？路径为 /posts/first-post
  • 打开 http://localhost:3000/posts/first-post

总结

  • 约定大于配置

Link快速导航

Link 对比 a标签的好处

  • Link跳转的页面不会刷新，用ajax请求页面内容 (first-post.js)
  • 不会请求重复的HTML/JS/CSS
  • 自动在页面插入新内容，删除旧内容
  • 因为省了很多请求和解析过程，所以速度很快

借鉴了 以前就有的技术 客户端导航 如 pjax技术

同构代码

代码同时运行在两端

  • 组件里 console.log(1)
  • 你会发现 Node控制台/浏览器 都会输出 console.log(1)

差异

  • 不是所有代码，都会运行，有些需要用户触发 比如 btn.click
  • 不是所有API都能用，比如 window 在 Node里报错

参考 pages/posts/first-post.js

全局配置

  • 自定义 Head
  • 全局 CSS
  • 局部 CSS

自定义 Head

  • 每个组件内你可以这样

如何修改全局 title 和 meta

  • 修改 pages/_app.js
  • 添加 <Head> 如果你想覆盖全局的 head，则在你的页面写head即可
  • export default function App 是每个组件的根组件
  • 页面切换 App不会销毁，App里面的组件回销毁
  • 可用 App 保留全局状态

修改了 _app.js需要重启才能生效

全局CSS

  • 在 styles/globals.css 里设置全局样式
  • 在pages/_app.js里引入 import '../styles/globals.css'
  • 其他地方不能 引入全局 css ,会报错

局部css

  • 可以引入 import styles from '../styles/Home.module.css'
  • 还可以jsx 写 css

index.js

模块化的css

  • 新建 first-post.module.css
  • 修改 pages/posts/first-post.js

如何引入文件不是相对路径

  • 项目根目录 新建 jsconfig.js

使用 sass

  • yarn add sass
  • 直接 .css文件修改为 .scss即可

静态资源

next推荐放在 public/里

不推荐资源放在 public里

  • public/logo.png
  • 页面里 <img src="/logo.png" />
  • 但是这样无法被缓存

正确姿势

  • 项目根目录新建 assets/images
  • 你不能直接这样,会报错
  • 你需要修改配置 next.config.js

配置一个img那么麻烦， next-images 插件

修改配置 next.config.js
